Introduction

What happens when you visit a Web site?

  1. All machines on the Internet use an Internet Protocol (IP) address to refer to each other, e.g.: 69.163.184.47
  2. User types a URL into a browser address bar or follows a link, e.g.:
    http://wdux1718.bbkweb.org/sessions/wdux09/
  3. Browser refers to Domain Name Servers to convert wdux1718.bbkweb.org into an IP address
  4. The IP address is for the server hosting the web site
    • What's a server? Hardware or software?

Introduction continued

  1. Browser sends a request for /sessions/wdux09/ to the server
  2. Web server receives the request and looks for an index file within the 'wdux09' directory within the 'sessions' directory
  3. HTML file (index.html) sent back to browser
  4. Browser reads the HTML and sends requests for any ancillary files to the server:
    • CSS
    • Images
    • JavaScript files

    Modern browsers can make 6-16 concurrent requests.

Web servers

Lots of different Web servers, e.g:

  • Apache - most popular
  • Nginx
  • Microsoft Internet Information Services (IIS)
  • Lighttpd

Choice of Web server depends on the operating system on the server, performance, functionality etc.

Client-side and server-side technology

  • Client-side: runs within the browser.
  • Server-side: runs on the server.
  • JavaScript (client-side) - some example uses:
    • Date pickers
    • Auto-completion
    • Show/hide content

Server-side technology

  • Server-side - example languages:
    • PHP
    • Python
    • C#
    • Java
    • JavaScript
    Example uses:
    • Online shopping
